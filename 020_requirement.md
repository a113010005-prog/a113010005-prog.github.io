## User Stories

寫至少 2 條。格式：`作為___，我想要___，以便___。`

範例（不要抄，自己想）：
- 作為記者，我想要一鍵把 Markdown 稿件發布到網站上，以便省去手動排版的時間。
- 作為編輯，我想要同一篇文章自動產出 FB 和 X 的版本，以便同步推廣。

## 思考題（選寫，但寫了加分）

1. 你的網站未來想放什麼內容？（新聞？作品集？筆記？）
2. 如果你每週要發 3 篇文章，從寫稿到上線的流程怎麼設計？哪些步驟可以自動化？
3. 靜態網站（Astro）和動態網站（WordPress）各適合什麼場景？你會怎麼選？
4. 你覺得「一稿多發」最大的挑戰是什麼？不同平台的讀者期待有什麼不同？
